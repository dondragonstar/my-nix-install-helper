# CONFIGS MASTER ‚Äî Hydragon2000's NixOS system

> **Source of truth**: `/etc/nixos/` (Nix flake + Home Manager)
> All user-level configs under `~/.config/` are *generated* or *templated* from
> `home.nix` by Home Manager. **Edit the flake, not the runtime files.**
> Runtime files (`~/.config/waybar/config.jsonc`, `~/.config/hypr/hyprland.conf`, etc.)
> get overwritten on `nixos-rebuild switch` ‚Äî so edit `/etc/nixos/home.nix` instead.

---

## A. SYSTEM-LEVEL CONFIG ‚Äî `/etc/nixos/` (Nix Flake)

| File | Purpose | Notes |
|---|---|---|---|
| `/etc/nixos/flake.nix` | Flake entry point; defines `hostname` + `username` vars, passes them to all modules via `specialArgs`. Has `wlctl` flake input for NetworkManager TUI. | **Edit hostname/username here on new machine** |
| `/etc/nixos/configuration.nix` | System config: bootloader, networking, GPU, Hyprland, PipeWire, SDDM autologin | GPU section marked `MACHINE-SPECIFIC` ‚Äî edit per hardware |
| `/etc/nixos/hardware-configuration.nix` | Auto-generated hardware config (generated by `nixos-generate-config`) | Read-only; regenerated on each machine. In `.gitignore` |
| `/etc/nixos/home.nix` | **User config**: git, SSH, GTK theme, cursor, Zsh, Hyprland, Waybar, all packages | Uses `{ username, hostname }` variables from flake |
| `/etc/nixos/theme.nix` | Theme system: waybar colors + cursor theme. Has 3 themes, `default` and `bibata-classic` are duplicates | **Duct tape** ‚Äî duplicate themes, AI-generated and never deduped |
| `/etc/nixos/waybar-config.jsonc` | Waybar layout JSON, sourced into `~/.config/waybar/config.jsonc` by home.nix | Edit this if you want waybar changes (not the runtime file) |
| `/etc/nixos/flake.lock` | Pinned dependency versions | Auto-generated |
| `/etc/nixos/repo/` | Git repo ‚Äî contains all files above + `README.md` + `sync.sh` + `gen-commit-msg.py` | **The portable flake ‚Äî clone this for a fresh install** |
| `/etc/nixos/repo/README.md` | Install instructions for a fresh NixOS system | Covers partitioning, install, and per-machine customization |

## C. DUCT TAPE / AI-BUILT-AND-NEVER-REFACTORED ITEMS

1. **`/etc/nixos/theme.nix`**: 3 themes where `default` and `bibata-classic` are identical. The `google-dot` theme only differs in cursor name. Clearly AI-generated once and never cleaned up.

2. **`~/.zshenv`**: Has `if [[ -z "${__HM_ZSH_SESS_VARS_SOURCED-}" ]]; then export __HM_ZSH_SESS_VARS_SOURCED=1; fi` with literally nothing inside the if block. The guard exists but guards nothing.

3. **`~/.config/fontconfig/conf.d/52-hm-default-fonts.conf`**: Valid XML with an empty `<fontconfig>` block. Does nothing.

4. **`~/.config/vlc/vlcrc`**: 4666 lines of all-commented-out defaults. VLC creates this on first launch and never prunes it.

5. **Waybar config duplication**: The source config is in `/etc/nixos/waybar-config.jsonc`, but it gets copied to `~/.config/waybar/config.jsonc`. The runtime file has slightly different formatting (no trailing commas) ‚Äî meaning someone edited the runtime file after deployment. On next `nixos-rebuild`, those changes vanish.

6. **`/etc/nixos/repo/` vs `/etc/nixos/`**: NixOS configs exist in TWO places ‚Äî `/etc/nixos/` (active) and `/etc/nixos/repo/` (git repo copy). This invites drift.

7. **nixpkgs Hyprland 0.55.2 built without Lua**: Despite Hyprland 0.55+ supporting Lua natively, nixpkgs compiles it without Lua support. Must use hyprlang (`.conf`) format. Check with `ldd $(which Hyprland) | grep lua` ‚Äî if no output, Lua is not available.

## E. CONFIG HIERARCHY (HOW THINGS CONNECT)

```
/etc/nixos/flake.nix
  ‚îú‚îÄ‚îÄ /etc/nixos/configuration.nix     (system: boot, GPU, Hyprland, PipeWire, SDDM)
  ‚îú‚îÄ‚îÄ /etc/nixos/hardware-configuration.nix  (auto-generated)
  ‚îî‚îÄ‚îÄ home-manager
        ‚îî‚îÄ‚îÄ /etc/nixos/home.nix         (user: git, SSH, GTK, Zsh, packages)
              ‚îú‚îÄ‚îÄ /etc/nixos/theme.nix  (theme vars ‚Üí waybar colors, cursor)
              ‚îú‚îÄ‚îÄ /etc/nixos/waybar-config.jsonc  (copied ‚Üí ~/.config/waybar/)
              ‚îú‚îÄ‚îÄ /etc/nixos/alacritty.toml       (copied ‚Üí ~/.config/alacritty/)
              ‚îî‚îÄ‚îÄ inline text blocks ‚Üí various ~/.config/ files
                  (hyprland.conf, impala/config.toml, waybar/style.css, etc.)
```

**Golden rule**: Edit `/etc/nixos/home.nix` for user-level changes, `/etc/nixos/configuration.nix` for system-level. **AI agents must NOT run nixos-rebuild** ‚Äî the human handles that after reviewing changes.

## G. THE SYNC LOOP (CRITICAL ‚Äî AI AGENTS MUST FOLLOW THIS)

The NixOS configs live in two places:
- **`/etc/nixos/`** ‚Äî the *live* configs NixOS actually reads
- **`/etc/nixos/repo/`** ‚Äî a git repo (`origin: git@github.com:dondragonstar/my-nix-install-helper.git`)

They **will diverge** if not synced. The sync script does:
```
/etc/nixos/  ‚îÄ‚îÄcopy‚îÄ‚îÄ‚Üí  /etc/nixos/repo/  ‚îÄ‚îÄcommit‚îÄ‚îÄ‚Üí  git push origin main
```

### Mandatory workflow for EVERY AGENT SESSION

```
1. Edit config files under /etc/nixos/ or ~/CONFIGS_MASTER.md
2. Run /etc/nixos/repo/sync.sh
   ‚Üí copies changed files into repo, commits, pushes to origin/main
   ‚Üí permitted without prompting by opencode's permission config
```

The script handles:
- `configuration.nix`, `flake.nix`, `flake.lock`, `home.nix`, `theme.nix`
- `waybar-config.jsonc`, `alacritty.toml`, `CONFIGS_MASTER.md`
- Excludes `hardware-configuration.nix` (auto-generated)

### AI commit messages (NEW ‚Äî 2026-07-05)

`sync.sh` now pipes the actual git diff to `qwen2.5-coder:3b` (local Ollama) to generate
**meaningful commit messages** describing what changed and why. No more `auto-sync: date` timestamps.

The flow:
```
edit config ‚Üí sync.sh ‚Üí copies files ‚Üí git diff | ollama ‚Üí AI writes commit msg ‚Üí push
```

- `sync.sh:41-43` ‚Äî sends diff to Ollama; falls back to stat-based msg if AI fails
- `gen-commit-msg.py` ‚Äî Python helper that calls Ollama API, lives in repo dir
