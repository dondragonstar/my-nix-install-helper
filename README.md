# Hydragon2000's NixOS Config

NixOS flake + Home Manager configuration featuring Hyprland, Waybar, NVIDIA Optimus PRIME, Ollama, and a custom theming system.

## Fresh Install (new machine)

```bash
# 1. Boot from NixOS USB installer

# 2. Partition & format (example: single ext4 + EFI)
#    Adjust device names to your hardware!
sudo parted /dev/nvme0n1 -- mklabel gpt
sudo parted /dev/nvme0n1 -- mkpart primary 512MB 100%
sudo parted /dev/nvme0n1 -- mkpart ESP fat32 1MB 512MB
sudo parted /dev/nvme0n1 -- set 2 esp on
sudo mkfs.ext4 -L nixos /dev/nvme0n1p1
sudo mkfs.fat -F 32 -n boot /dev/nvme0n1p2

# 3. Mount
sudo mount /dev/disk/by-label/nixos /mnt
sudo mkdir -p /mnt/boot
sudo mount /dev/disk/by-label/boot /mnt/boot

# 4. Generate hardware config (detects disks, kernel modules, etc.)
sudo nixos-generate-config --root /mnt

# 5. Clone this repo onto the live system
#    (git is available on the installer ISO)
sudo git clone https://github.com/dondragonstar/my-nix-install-helper.git /mnt/etc/nixos

# 6. Overwrite the generated hardware config into the repo
#    (it was generated in step 4 at /mnt/etc/nixos/hardware-configuration.nix)
#    No action needed — the repo's flake.nix already imports it.

# 7. Customize for this machine — edit these files:
#    flake.nix           → change hostname and username (top of the file)
#    configuration.nix   → change GPU section (marked "MACHINE-SPECIFIC")
#    home.nix            → change git user.name / user.email

# 8. Install!
sudo nixos-install --flake /mnt/etc/nixos#your-new-hostname

# 9. Reboot, set a password for your user, and you're done
sudo passwd hydragon2000
sudo reboot
```

## Machine-specific things you **must** change per system

| What | File | Why |
|---|---|---|
| `hostname` + `username` | `flake.nix` (top `let` block) | Controls system hostname, user account, and home dir |
| GPU / graphics driver | `configuration.nix` (section marked `MACHINE-SPECIFIC`) | NVIDIA, AMD, or Intel-only each need different config |
| `hardware-configuration.nix` | auto-generated by `nixos-generate-config` | Disk UUIDs, kernel modules — **never edit manually** |
| git name/email | `home.nix` | Your git identity |
| SSH keys | `home.nix` → `programs.ssh.matchBlocks` | Point to keys that exist on this machine |
| time zone | `configuration.nix` | Set your `time.timeZone` |

## After install — day-to-day use

Edit a config, then sync to GitHub:

```bash
sudo nixos-rebuild switch --flake /etc/nixos#hydragon2000-pc
/etc/nixos/repo/sync.sh    # copies changes into repo + commits + pushes
```

Aliases (set in `.zshrc`): `rebuild` and `drybuild`.

## How the sync loop works

- `/etc/nixos/` — live configs NixOS reads
- `/etc/nixos/repo/` — git repo that mirrors the live configs
- `sync.sh` copies live files → repo → generates AI commit message via `qwen2.5-coder:3b` (Ollama) → pushes

## Notes

- `hardware-configuration.nix` is in `.gitignore` — each machine generates its own
- The GPU config assumes NVIDIA RTX 2050 + Intel Optimus PRIME offload. Replace for other hardware.
- Ollama is configured with `ollama-cuda` (NVIDIA CUDA backend). Remove `-cuda` or change to `ollama-rocm` for AMD.
